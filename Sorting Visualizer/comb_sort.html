Java


public static void combSort(int[] arr) {
  int n = arr.length;
  int gap = n;
  boolean swapped = true;
  while (gap != 1 || swapped) {
    gap = Math.max(1, (gap * 10) / 13);
    swapped = false;
    for (int i = 0; i + gap < n; i++) {
      if (arr[i] > arr[i + gap]) {
        int t = arr[i]; arr[i] = arr[i + gap]; arr[i + gap] = t;
        swapped = true;
      }
    }
  }
}

      
C++


void combSort(int arr[], int n) {
  int gap = n;
  bool swapped = true;
  while (gap != 1 || swapped) {
    gap = max(1, (gap * 10) / 13);
    swapped = false;
    for (int i = 0; i + gap < n; i++) {
      if (arr[i] > arr[i + gap]) {
        swap(arr[i], arr[i + gap]);
        swapped = true;
      }
    }
  }
}

      

    

    

      
Python


def comb_sort(arr):
    n = len(arr)
    gap = n
    swapped = True
    while gap != 1 or swapped:
        gap = max(1, int(gap / 1.3))
        swapped = False
        for i in range(n - gap):
            if arr[i] > arr[i + gap]:
                arr[i], arr[i + gap] = arr[i + gap], arr[i]
                swapped = True

Comb Sort: Complexity & Key Points

  
    
    
  • Best case: O(n \log n) (almost sorted, few gaps)
    • 
    
  • Worst/Average case: ~O(n^2) (like bubble, little improvement for bad inputs)
    • 
    
  • Space: O(1) (in-place)
    • 
    
  • Generalization of bubble sort. Removes "turtles" using gap > 1.
    • 
    
  • Gap starts at array size and shrinks by 1.3x every pass, then finishes with bubble sort
    • 
    
  • More efficient than bubble sort, but not commonly used in real applications
