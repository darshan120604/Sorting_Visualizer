Learn Sorting Concepts

Complexity:
  • Time Complexity:
    • O(n^2): slow for big n (Bubble, Selection, Insertion, Stooge, Cycle)
    • O(n log n): fast & scalable (Merge, Heap, Quick on average, Tim)
    • O(n): for limited numeric inputs (Counting, Bucket, Radix; only when input range is small/moderate!)
  • Space Complexity:
    • In-place (O(1)): sorts without major extra memory (Bubble, Selection, Insertion, Quick, Heap, Cycle)
    • Extra Array (O(n)): uses new arrays (Merge, Counting, Bucket, Radix, Tim)
Key Definitions:
  • Stable: does not rearrange equal elements out of their original order (Merge, Insertion, Bubble, Tim, Counting, etc.).
  • In-place: sorts by overwriting the input without significant helper arrays.
  • Comparison-based: only compares using <, ==, >. (Bubble, Selection, Quick, etc.)
  • Non-comparison-based: sorts by value, not comparison. (Counting, Radix, Bucket)

Visual Approach: Best way to learn: step through or animate — observe splits, swaps, merges, and final sorted state. Try worst and best cases!
